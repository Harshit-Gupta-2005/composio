Composio relies on a small set of infrastructure providers to host the platform, execute your tool calls, and deliver triggers. The **authoritative, current list of our sub-processors** — with each provider's purpose and region — lives in the [Composio Trust Center](https://trust.composio.dev/subprocessors). Always refer to that list for specific vendors and regions.

This page explains, at a category level, where your data goes so you can reason about exposure.

## How your data flows

- **Tool calls** are made to the **destination provider** you are integrating with (for example, your CRM or email service). That provider receives the request and returns the response.
- **Execution and logging** run on Composio's infrastructure. Whether the request and response payloads are stored is controlled by your [Log storage setting](/docs/security/data-retention) — set it to "Don't store data" to keep only the audit record.
- **Files** in tool calls may pass through object storage while a call runs.
- **Code execution** runs in an isolated sandbox, only when you use the workbench or code execution.
- **Triggers** are delivered to the destinations you configure, in real time.
- **Tool discovery** sends your natural-language search query (not your execution payloads) to a model and embeddings provider to find relevant tools.

## What is not sent

Operational monitoring and billing systems receive metadata and usage information, not your tool-call payloads.

## Data residency

Composio's primary processing region is the United States (AWS `us-east-1`).

For the current list of sub-processors and our data-processing terms, see the [Composio Trust Center](https://trust.composio.dev/subprocessors).
